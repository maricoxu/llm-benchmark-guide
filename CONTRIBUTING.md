# 贡献指南

感谢你对LLM Benchmark Guide项目的关注！我们欢迎任何形式的贡献。

## 🤝 如何贡献

### 报告问题

如果你发现了问题或有改进建议，请在 [Issues](https://github.com/yourusername/llm-benchmark-guide/issues) 中报告。

### 改进文档

- 修正错误
- 改进现有文档
- 添加新的评测集文档
- 完善使用指南

### 提交代码

- 添加评测工具示例
- 改进评测脚本
- 添加自动化工具

## 📝 贡献流程

1. **Fork 仓库**: Fork本仓库到你的GitHub账号
2. **创建分支**: 创建新的分支进行修改
3. **提交更改**: 提交你的更改
4. **创建PR**: 创建Pull Request

## 📋 文档规范

### 评测集文档格式

每个评测集文档应包含：

- **基本信息表格**: 测试条目数、输入/输出长度等
- **评测集介绍**: 目的、特点、适用场景
- **资源链接**: GitHub、论文、数据集链接
- **使用指南**: 如何使用评测集
- **最佳实践**: 评测方法和注意事项
- **常见问题**: FAQ

### 代码规范

- 使用清晰的变量名
- 添加必要的注释
- 遵循PEP 8（Python）或其他语言的代码规范

## ✅ 检查清单

提交PR前，请确保：

- [ ] 文档格式正确
- [ ] 链接有效
- [ ] 代码可以运行
- [ ] 遵循项目规范

**感谢你的贡献！** 🙏
